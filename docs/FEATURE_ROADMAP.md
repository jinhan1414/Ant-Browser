# 功能路线图

本文档记录待开发的功能列表，按优先级和类别分组，供后续选择实现。

---

## 一、实用性增强

### 1.1 批量操作
- **描述**：支持在实例列表中多选，批量执行启动/停止/删除操作
- **实现要点**：
  - 列表添加复选框列
  - 顶部工具栏显示已选数量和批量操作按钮
  - 批量启动需考虑并发控制，避免同时启动过多实例
- **复杂度**：中
- **优先级**：高

### 1.2 实例分组（文件夹）
- **描述**：除标签外，支持文件夹式层级分组，类似浏览器书签文件夹
- **实现要点**：
  - 新增 Group 表存储分组信息
  - 实例表添加 groupId 字段
  - 左侧树形导航或下拉筛选
- **复杂度**：中
- **优先级**：中

### 1.3 导入/导出配置
- **描述**：将实例配置导出为 JSON 文件，支持导入恢复
- **实现要点**：
  - 导出：选择实例 → 生成 JSON（含指纹、代理、标签等）
  - 导入：解析 JSON → 创建新实例（生成新 ID 和指纹种子）
  - 支持批量导入
- **复杂度**：低
- **优先级**：高

### 1.4 代理轮换
- **描述**：实例启动时自动从代理池中轮换选择代理
- **实现要点**：
  - 实例配置新增"代理轮换"选项
  - 启动时按策略选择代理（顺序/随机/最少使用）
  - 记录每个代理的使用次数
- **复杂度**：中
- **优先级**：中

---

## 二、自动化相关

### 2.1 定时任务
- **描述**：支持定时启动/停止实例，或定时打开指定 URL
- **实现要点**：
  - 新增定时任务表（cron 表达式或简单时间配置）
  - 后台调度器执行任务
  - 任务类型：启动实例、停止实例、打开 URL、执行脚本
- **复杂度**：高
- **优先级**：低

### 2.2 启动脚本
- **描述**：实例启动后自动执行预设操作（打开网页、等待、点击等）
- **实现要点**：
  - 实例配置新增"启动脚本"字段
  - 脚本格式：简单 DSL 或 JSON 配置
  - 通过 CDP 协议执行操作
- **复杂度**：高
- **优先级**：低

### 2.3 健康检查
- **描述**：定期检测运行中实例是否正常，异常时自动重启
- **实现要点**：
  - 后台定时检查进程状态和 CDP 连接
  - 检测到异常后尝试重启
  - 记录异常日志
- **复杂度**：中
- **优先级**：中

---

## 三、监控与统计

### 3.1 运行时长统计
- **描述**：记录每个实例的累计运行时间
- **实现要点**：
  - 实例表新增 totalRuntime 字段
  - 启动时记录 startTime，停止时累加时长
  - 列表和详情页展示运行时长
- **复杂度**：低
- **优先级**：中

### 3.2 代理流量统计
- **描述**：统计每个实例通过代理的流量消耗
- **实现要点**：
  - 需要代理工具（xray/sing-box）支持流量统计 API
  - 定期采集并存储到数据库
  - 仪表盘展示流量趋势
- **复杂度**：高
- **优先级**：低

### 3.3 操作日志
- **描述**：记录实例的启动/停止/配置变更等操作历史
- **实现要点**：
  - 新增操作日志表（时间、操作类型、实例 ID、详情）
  - 关键操作时写入日志
  - 详情页展示操作历史
- **复杂度**：低
- **优先级**：中

---

## 四、指纹相关

### 4.1 指纹模板
- **描述**：保存常用指纹配置为模板，创建实例时快速应用
- **实现要点**：
  - 新增指纹模板表
  - 编辑页支持"保存为模板"和"从模板加载"
  - 模板管理页面（增删改查）
- **复杂度**：中
- **优先级**：中

### 4.2 指纹检测
- **描述**：启动实例后自动访问指纹检测网站并记录结果
- **实现要点**：
  - 配置检测网站列表（browserleaks、pixelscan 等）
  - 启动后自动打开检测页面
  - 可选：截图保存检测结果
- **复杂度**：中
- **优先级**：低

---

## 五、体验优化

### 5.1 快捷键支持
- **描述**：键盘快捷键操作实例
- **实现要点**：
  - 全局快捷键：Ctrl+N 新建、Ctrl+R 刷新
  - 列表快捷键：Enter 启动、Delete 删除
  - 使用 Wails 的快捷键 API
- **复杂度**：低
- **优先级**：低

### 5.2 托盘最小化
- **描述**：最小化到系统托盘，后台运行
- **实现要点**：
  - 使用 Wails 的系统托盘 API
  - 托盘菜单：显示窗口、退出
  - 关闭窗口时最小化到托盘而非退出
- **复杂度**：低
- **优先级**：中

### 5.3 主题切换
- **描述**：支持深色/浅色主题切换
- **实现要点**：
  - CSS 变量已支持主题
  - 添加主题切换按钮
  - 保存用户偏好到 localStorage
- **复杂度**：低
- **优先级**：低

### 5.4 多语言支持
- **描述**：支持中英文切换
- **实现要点**：
  - 使用 i18n 库（react-i18next）
  - 提取所有文案到语言文件
  - 设置页添加语言切换
- **复杂度**：中
- **优先级**：低

---

## 六、高级功能

### 6.1 实例同步
- **描述**：多台机器间同步实例配置
- **实现要点**：
  - 云端存储配置（需要后端服务）
  - 或通过文件同步（导入/导出增强版）
- **复杂度**：高
- **优先级**：低

### 6.2 API 扩展
- **描述**：扩展 LaunchCode API，支持更多远程控制操作
- **实现要点**：
  - 新增 API：获取实例状态、修改配置、获取 Cookie 等
  - 完善 API 文档
  - 添加 API 认证机制
- **复杂度**：中
- **优先级**：中

### 6.3 插件系统
- **描述**：支持用户自定义插件扩展功能
- **实现要点**：
  - 定义插件接口和生命周期
  - 插件加载和管理
  - 插件市场（远期）
- **复杂度**：高
- **优先级**：低

---

## 推荐实现顺序

根据实用性和复杂度，建议按以下顺序实现：

1. **导入/导出配置** - 低复杂度，高实用性
2. **批量操作** - 中复杂度，高实用性
3. **运行时长统计** - 低复杂度，中实用性
4. **操作日志** - 低复杂度，中实用性
5. **托盘最小化** - 低复杂度，中实用性
6. **指纹模板** - 中复杂度，中实用性
7. **健康检查** - 中复杂度，中实用性
8. **代理轮换** - 中复杂度，中实用性
